## Соглашения

### Стиль документации

- **Markdown** — все документы в формате Markdown
- **Русский язык** — основной язык документации (код на английском)
- **Примеры кода** — всегда с комментариями и пояснениями
- **Актуальность** — дата последнего обновления в конце каждого документа

### Обновление документации

При внесении изменений в проект:

1. Обновите соответствующий раздел документации
2. Проверьте cross-references на актуальность
3. Добавьте дату обновления в конец документа

### Добавление новой документации

1. Создайте файл в `/docs/`
2. Добавьте ссылку в этот файл (INDEX.md)
3. Обновите CLAUDE.md при необходимости
4. Добавьте ссылку в README.md если документ важный
